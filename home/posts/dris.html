title = Data Storage, Replication, and Integrity Syllabus ---

Reading list designed to help programmers understand the domain of storage, replication, and integrity. There are basically no modern books on the topic as a whole.

THIS IS WORK IN PROGRESS. [Send me](mailto:phil@eatonphil.com) suggestions.

### Part 1: Data structures

Reading list:

  • B-Tree
    • [More Modern B-Tree Techniques](https://www.nowpublishers.com/article/Details/DBS-070)
    • [An Introduction to B epsilon trees and Write Optimization](https://www.usenix.org/system/files/login/articles/login_oct15_05_bender.pdf)
    • [B-Tree Visualization](https://www.cs.usfca.edu/~galles/visualization/BTree.html)
    • [B-Trees Are Back: Engineering Fast and Pageable Node Layouts](https://www.cs.cit.tum.de/fileadmin/w00cfj/dis/papers/btrees-are-back.pdf)
  • LSM Tree
    • [LSM-based Storage Techniques: A Survey](https://arxiv.org/pdf/1812.07527)
    • [A Critical Survey on Log-Structured Merge Trees](https://www.researchgate.net/profile/Sophianos-Vasileios-2/publication/385097991_A_Critical_Survey_on_Log-Structured_Merge_Trees/links/671666e968ac304149a58478/A-Critical-Survey-on-Log-Structured-Merge-Trees.pdf)
    • [Towards Flexibility and Robustness of LSM Trees](https://arxiv.org/pdf/2311.10005)
  • [Btree vs LSM](https://github.com/wiredtiger/wiredtiger/wiki/Btree-vs-LSM)
  • [Revisiting B+-tree vs. LSM-tree](https://www.usenix.org/publications/loginonline/revisit-b-tree-vs-lsm-tree-upon-arrival-modern-storage-hardware-built)

Key topics: LSM Tree, B-Tree

### Part 2: Dealing with failure

Reading list:

  • [Enhancing File System Integrity Through Checksums](https://citeseerx.ist.psu.edu/document?repid=rep1&type=pdf&doi=c15fac2198414e96db38d176aa22d52293264eac)
  • [End-to-end Data Integrity for File Systems: A ZFS Case Study](https://www.usenix.org/legacy/event/fast10/tech/full_papers/fast10proceedings.pdf#page=37)
  • [Can Applications Recover from fsync Failures?](https://www.usenix.org/system/files/atc20-rebello.pdf)
  • [An Analysis of Latent Sector Errors in Disk Drives](https://research.cs.wisc.edu/adsl/Publications/latent-sigmetrics07.pdf)
  • [Gray Failure: The Achilles' Heel of Cloud-Scale Systems](https://www.microsoft.com/en-us/research/wp-content/uploads/2017/06/paper-1.pdf)
  • [The Tail at Store: A Revelation from Millions of Hours of Disk and SSD Deployments](https://www.usenix.org/system/files/conference/fast16/fast16-papers-hao.pdf)
  • [The Tail at Scale](https://courses.cs.duke.edu//cps296.4/fall13/838-CloudPapers/dean_longtail.pdf)
  • [An Analysis of Data Corruption in the Storage Stack](https://www.usenix.org/legacy/events/fast08/tech/full_papers/bairavasundaram/bairavasundaram.pdf)
  • [A Study of SSD Reliability in Large Scale Enterprise Storage Deployments](https://www.usenix.org/system/files/fast20-maneas.pdf)

Key topics: torn writes, torn reads, misdirected writes, misdirected reads, use of checksums, fsync

### Part 3: IO models

Reading list:

  • [Understanding Modern Storage APIs: A systematic study of libaio, SPDK, and io_uring](https://atlarge-research.com/pdfs/2022-systor-apis.pdf)
  • [What Modern NVMe Storage Can Do, And How To Exploit It: High-Performance I/O for High-Performance Storage Engines](https://vldb.org/pvldb/vol16/p2090-haas.pdf)
  • [Asynchronous I/O — With Great Power Comes Great Responsibility](https://arxiv.org/pdf/2411.16254)
  • [Characteristics of multithreading models for high-performance IO driven network applications](https://arxiv.org/pdf/0909.4934)

Key topics: thread-per-core, io_uring, epoll, aio, spdk

### Part 4: Replication

Reading list:

  • [Data Replication Design Spectrum](https://transactional.blog/blog/2024-data-replication-design-spectrum)
  • Quorums and leaderless consensus
    • [Cassandra \- A Decentralized Structured Storage System](https://www.cs.cornell.edu/projects/ladis2009/papers/lakshman-ladis2009.pdf)
    • [EPaxos Revisited](https://www.usenix.org/system/files/nsdi21-tollman.pdf)
    • [Fast General Purpose Transactions in Apache Cassandra](https://cwiki.apache.org/confluence/download/attachments/188744725/Accord.pdf?version=1&modificationDate=1630847737000&api=v2)
  • Reconfiguration
    • [Object Storage on CRAQ](https://www.usenix.org/legacy/event/usenix09/tech/full_papers/terrace/terrace.pdf)
      • [6.824 FAQ for Object Storage on CRAQ](http://nil.lcs.mit.edu/6.824/2020/papers/craq-faq.txt)
    • [FoundationDB: A Distributed Unbundled Transactional Key Value Store](https://www.foundationdb.org/files/fdb-paper.pdf)
    • [Taurus Database: How to be Fast, Available, and Frugal in the Cloud](https://arxiv.org/pdf/2412.02792)
    • [PacificA: Replication in Log-Based Distributed Storage Systems](https://www.microsoft.com/en-us/research/wp-content/uploads/2008/02/tr-2008-25.pdf)
  • Leader-based consensus
    • [In Search of an Understandable Consensus Algorithm (Extended Version)](https://raft.github.io/raft.pdf)
      • [Consensus: Bridging Theory and Practice](https://web.stanford.edu/~ouster/cgi-bin/papers/OngaroPhD.pdf)
      • [Paxos vs Raft: Have we reached consensus on distributed consensus?](https://arxiv.org/pdf/2004.05074)
      • [Paxos Made Live \- An Engineering Perspective](https://www.cs.utexas.edu/~lorenzo/corsi/cs380d/papers/paper2-1.pdf)
      • [A TiKV Source Code Walkthrough – Raft Optimization](https://www.pingcap.com/blog/optimizing-raft-in-tikv/)
      • [Simplifying Redpanda Raft implementation](https://www.redpanda.com/blog/simplifying-raft-replication-in-redpanda)
    • [Viewstamped Replication Revisited](https://pmg.csail.mit.edu/papers/vr-revisited.pdf)

Key topics: raft, paxos, chain replication, viewstamped replication
